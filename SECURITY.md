# 安全与合规说明

本仓库用于内部 AI 辅助研发培训。所有课程材料、练习项目、日志、案例和知识库样例都必须满足安全与合规要求。

## 一、禁止提交的内容

不得提交：

- 真实客户数据。
- 真实生产日志中的敏感字段。
- 账号、密码、token、API key、证书、私钥。
- 内部系统地址、端口、账号体系等敏感环境信息。
- 未授权公开的产品代码。
- 未脱敏的 DTS、故障单、客户问题单。
- 完整标准答案和隐藏测试。

## 二、允许提交的内容

可以提交：

- 人工构造的训练数据。
- 已脱敏、已裁剪的日志片段。
- 简化后的数据库训练项目。
- 与真实项目结构相似但不包含敏感实现的示例代码。
- 经过授权的公开开源代码片段。
- 用于教学的假 DTS、假故障单和模拟压测结果。

## 三、脱敏要求

提交日志、DTS 摘要或故障案例前，应至少处理：

| 类型 | 处理方式 |
|---|---|
| 用户名、账号 | 替换为 `user_xxx` |
| IP、域名、内部 URL | 替换为 `host_xxx` 或 `example.internal` |
| 客户名称 | 替换为 `customer_xxx` |
| 路径 | 删除个人目录或内部项目真实路径 |
| 密钥、token | 删除，不要替换后提交 |
| 时间戳 | 如无必要可模糊到日期级别 |

## 四、标准答案管理

训练仓不应直接暴露完整标准答案。建议：

- 标准答案放在讲师私有仓。
- 隐藏测试放在讲师私有仓或内部评分平台。
- 课后参考答案如需公开，应单独 review 后发布。
- 公开参考答案应避免包含对隐藏测试的直接提示。

## 五、使用外部模型的注意事项

如果练习需要调用外部 LLM 或 Coding Agent：

1. 不要上传敏感代码或内部数据。
2. 不要在 prompt 中包含密钥、内部地址、客户信息。
3. 遵守公司关于黄区、蓝区和外部服务使用的规定。
4. 对需要脱敏的材料，先脱敏再提供给模型。
5. 对模型输出进行人工复核，不直接当作事实。

## 六、PR 安全检查

每个 PR 合入前应检查：

- 是否包含疑似密钥。
- 是否包含真实客户或内部系统信息。
- 是否包含未脱敏日志。
- 是否暴露标准答案。
- 是否引入新的外部依赖或下载脚本。

如发现安全风险，应先移除风险内容，再重新提交 review。

## 七、问题上报

如果发现仓库中存在敏感信息或不应公开的材料，请立即通知仓库 owner 和课程负责人，并暂停相关材料发布。
