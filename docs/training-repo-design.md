# 培训仓设计说明

本文档用于向课程负责人、部门领导和主讲人说明 AI 辅助研发实战培训仓的设计方案。

## 一、设计目标

培训仓需要同时服务三类人：

1. 学员：能够提前一周获取材料，独立完成课前练习，并带着问题参加课堂。
2. 讲师：能够按统一结构发布课程材料、练习和验证脚本。
3. 管理者：能够检查课程是否按时发布、是否具备实战性、是否可持续沉淀。

因此，本仓库采用“课程材料 + 独立实战练习 + 自动化验证 + 统一规范”的结构。

## 二、为什么每节课一个目录

第一期培训不建议把 lesson 和 case 拆成两个独立体系。原因是：

- 学员只需要进入一个课程目录，就能看到 PPT、讲义、练习和验证脚本。
- 讲师维护成本更低，不需要在多个目录之间维护映射关系。
- 每节课可以独立发布，不依赖其他课程进度。
- 更适合课程试点阶段快速迭代。

后续如果某个练习被多节课复用，可以再抽取为公共 case 或共享练习项目。

## 三、推荐目录结构

```text
lessons/Lxx-topic/
├── README.md
├── lesson.yaml
├── slides/
├── docs/
├── exercise/
└── archive/
```

各目录职责如下：

| 路径 | 职责 |
|---|---|
| `README.md` | 学员入口，说明课程目标、课前准备、课堂安排、练习入口 |
| `lesson.yaml` | 机器可读的课程元数据，用于发布检查和统计 |
| `slides/` | 课程 PPT 源文件和导出的 PDF |
| `docs/` | 讲义、FAQ、讲师备注等文档 |
| `exercise/` | 本节课练习项目、任务说明、测试和验证脚本 |
| `archive/` | 课堂问题、优秀作业、课后补充材料 |

## 四、课程设计原则

每节课应围绕一个真实研发任务，而不是围绕一个孤立 AI 功能点。

推荐课程主线：

| 课次 | 主题 |
|---:|---|
| 0 | Coding Agent 安装配置与基础体验 |
| 1 | 从已有代码仓开始：Coding Agent 研发最小闭环 |
| 2 | 接手陌生代码仓：建立 AI 可用的项目工作台 |
| 3 | 从需求到代码：AI 辅助实现一个小功能 |
| 4 | AI 辅助代码检视与重构 |
| 5 | AI 辅助测试设计与测试代码生成 |
| 6 | AI 辅助问题定位：日志、编译、压测与并发异常 |
| 7 | 构建 Skill：把高频研发流程固化下来 |
| 8 | 构建知识库：让 AI 用上团队经验和历史案例 |
| 9 | Capstone：真实场景综合实战 |

## 五、上下文控制作为贯穿要求

上下文控制不单独作为一节课，而是每节课的共同评分项。

每次练习建议要求学员提交 `context.md`，至少说明：

- 本次任务是什么。
- 给 Coding Agent 提供了哪些上下文。
- 明确限制它不要改哪些内容。
- 使用什么方法验证结果。
- Agent 出现了哪些错误判断，人工如何纠正。
- 哪些内容可以后续沉淀成 skill 或知识库。

通用模板见 [shared/templates/context.template.md](../shared/templates/context.template.md)。

## 六、验证设计

每节课的练习都应提供统一验证入口：

```bash
cd lessons/Lxx-topic/exercise
./validators/validate.sh
```

验证内容可包括：

- 源码是否能构建。
- 单元测试或回归测试是否通过。
- 必要文档是否存在。
- 文档结构是否符合模板。
- 输出报告是否包含关键字段。
- 是否存在明显敏感信息。

验证脚本不要求完全判断答案优劣，但必须能检查基础交付质量。

## 七、答案管理

训练仓不应直接暴露完整标准答案。建议采用：

1. 学员仓：课程材料、练习、可见测试、验证脚本。
2. 讲师私有仓：标准答案、隐藏测试、评分细则。

如果需要课后公开参考答案，建议在课程结束后统一归档到专门目录或单独 release。

## 八、发布流程

建议采用如下节奏：

| 时间 | 动作 |
|---|---|
| T-14 天 | 讲师提交课程材料和练习初稿 |
| T-10 天 | 技术 reviewer 完成第一轮 review |
| T-7 天 | 合入主分支，通知学员预习 |
| T-2 天 | 收集问题，更新 FAQ |
| T 日 | 课堂讲解、答疑、讲题 |
| T+1 天 | 归档课堂问题，补充 FAQ |
| T+7 天 | 可选发布参考答案或优秀作业 |

## 九、质量标准

一节课可以合入的最低标准：

- 学员能独立找到学习入口。
- 练习任务清晰，预计 2 到 4 小时可完成。
- 验证脚本能在本地运行。
- 不包含敏感信息和标准答案。
- 讲义、PPT、FAQ、练习说明齐全。
- 课程目标和产出与总课表一致。

## 十、后续演进

第一期建议保持结构简单。后续可按实际情况演进：

- 将复用率高的练习抽取到公共 case。
- 增加仓库级 CI 检查。
- 增加隐藏测试和自动评分平台。
- 建立优秀作业归档区。
- 建立团队 rules、skills、knowledge 的版本治理机制。
