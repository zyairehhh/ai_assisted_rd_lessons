# L00 课前讲义：Coding Agent 安装配置与基础体验

## 1. 本课要解决什么

后续课程都会要求学员提前完成实战练习。如果工具无法启动、模型无法访问、代理配置不正确，正式课程会被环境问题拖慢。

因此 L00 只解决一件事：让每个人都能完成一次最小可用体验。

## 2. 推荐准备的工具

请根据团队要求至少准备一种：

- opencode
- Cursor
- Codex

如果你安装了多种工具，可以记录它们在同一个任务上的差异。

## 3. 常见配置项

| 配置项 | 说明 |
|---|---|
| API Key | 模型服务访问凭证，不要提交到仓库 |
| Base URL | 模型服务入口，按团队提供的信息配置 |
| Model | 默认使用的模型，不同模型能力和成本不同 |
| Proxy | 内网或跨区访问时可能需要 |
| Timeout | 大代码仓或长任务可能需要更长超时 |

## 4. 基础 prompt 形态

建议从以下几类开始体验：

```text
请阅读这个文件，说明它的主要职责和关键函数。
```

```text
请先不要修改代码，先给出你准备查看哪些文件以及原因。
```

```text
请根据测试失败信息定位可能原因，列出假设和验证方法。
```

```text
请 review 这段改动，只指出可能影响正确性、兼容性、性能或测试充分性的问题。
```

## 5. Skills 基础体验

Skill 是把固定流程沉淀成可复用说明的一种方式。L00 只要求完成下载或安装体验，不要求自己编写 skill。

如果由于权限或网络原因无法下载，请在 `agent_setup_report.md` 中记录原因和截图或命令输出摘要。

## 6. 安全要求

不要把以下内容发给外部模型或提交到仓库：

- API key、token、证书、私钥
- 真实客户数据
- 未脱敏日志
- 内部系统地址
- 未授权公开的产品代码

## 7. 后续课程共同要求

每节课练习都需要维护 `context.md`。它不是形式文档，而是用来训练你控制上下文：

- 给 AI 什么材料
- 不给 AI 什么材料
- 限制它不要做什么
- 如何验证它的结果
- 它错在哪里，你如何纠正
