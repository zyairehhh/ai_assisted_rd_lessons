# L01 FAQ

## Q1: 初始验证失败是不是环境问题？

不一定。本练习初始状态故意包含一个 bug，验证脚本失败是预期现象。你需要阅读失败信息并修复代码。

## Q2: 可以修改测试吗？

可以补充测试，但不能删除或降低已有测试要求。不要通过修改测试来绕过 bug。

## Q3: 可以让 AI 一次性改很多文件吗？

不建议。本课重点是控制修改范围。请优先限制在和 bug 相关的代码文件中。

## Q4: 如果 agent 判断错了怎么办？

请在 `context.md` 中记录错误判断、纠正方式和证据。这是本课训练的一部分。

## Q5: validation_report.md 写什么？

建议包含：

- 初始失败现象
- 根因分析
- 修改内容
- 验证命令
- 验证结果
- 剩余风险
