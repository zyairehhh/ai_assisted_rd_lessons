# L01 讲师备注

## 讲师目标

本课不要追求 bug 本身复杂。重点是让学员第一次完整走通 Coding Agent 研发闭环。

## 建议讲法

1. 展示初始测试失败。
2. 演示一个坏 prompt，让 agent 直接修，观察它可能扩大范围。
3. 演示一个好 prompt，让 agent 先读任务、读测试、列假设。
4. 强调修改范围和验证入口。
5. 展示如何写 `context.md` 和 `validation_report.md`。

## 练习 bug

MiniDB 当前对 `LIMIT 0` 的处理是错误的。正确行为应返回空结果。

## 课堂讨论

可以引导学员讨论：

- 哪些上下文是必要的。
- 哪些上下文是多余的。
- agent 有没有试图修改测试。
- 是否有学员做了无关重构。
